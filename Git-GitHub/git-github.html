Git e GitHub

O que é Git?

O Git é um sistema de controle de versão que, é uma máquina do tempo muito rápida e é um robô de integração muito competente.

Git grava as mudanças feitas em nossos arquivos ao longo do tempo, por meio de "snapshots".

Funciona como uma máquina do tempo para nosso código. Permitando o acompanhamento das alterações desde as versões mais antigas.

Tornando possível detectar e mesclar alterações nos mesmos arquivos, além de identificar conflitos, tudo de maneira automática, é conhecido como sistema de controle de versão

Foi criado em 2005 por Linus Torvalds, o mesmo criador do Linux

O que é GitHub?

O GitHub é uma aplicação Web que possibilita a hospedagem de repositórios Git, além de servir como uma rede social para programadores.

Foi criado em 208, diversos projetos de código aberto importantes são hospedados no GitHub.

Instalando e configurando o Git

Para fazer o download do git acessamos o seguinte site: http://msysgit.github.io/

A instalação é bastante simples. Escolha as opções padrão. Serão instalados alguns programas, sendo o mais importante o Git Bash, que permite que o Git seja executado pela linha de comando no Windows. Dê duplo clique no ícone do Git Bash e será aberto um terminal, com o seguinte prompt na linha de comando: $ Esse prompt será seu amigo a partir de agora. Não tenha medo! Sempre que falarmos de terminal, estaremos falando do Git Bash.

Instalando no Mac

Baixe a última versão do instalador gráfico do Git para Mac OS X a partir do link: https://code.google.com/p/git-osx-installer/downloads Abra um terminal e prepare-se para utilizar o Git!

Instalando no Linux

Para instalar o Git no Ubuntu, ou em uma outra distribuição baseada em Debian, execute em um terminal: $ sudo apt-get install git No Fedora, utilize: $ sudo yum install git Para as demais distribuições do Linux, veja o comando em: http:// git-scm.com/download/linux

Configurações Básicas

É importante nos identificarmos para o Git, informando nosso nome e e-mail. Em um terminal, execute os comandos a seguir:
$ git config --global user.name "Fulano da Silva" $ git config --global user.email fulanodasilva.git@gmail.com

Comandos mais utilizados

A maneira mais comum de usar Git é pela linha de comando, acessível através de um terminal. É o jeito que a maior parte dos bons profissionais do mercado utiliza o Git e será nossa escolha nesse livro.

git init:Para transformar o diretório atual em um repositório do Git.

git status:Para ver a situação dos arquivos no repositório Git.

git add: Para que o arquivo seja rastreado e seja adicionado a fase de stage(pronto para ser gravado).

git commit-m "Mensagem" : Para gravar(comitar) as alterações no repositório.

git commit -am "Mensagem": Com este comando juntamos o comando git add e git commit. Novos arquivos precisam de git add.

git logPara ver os commits que foram feitos.

git log --onelinePara ver todos os commits que foram feitos.

git log --statPara ver todos os detalhes dos commits que foram feitos.

git remote add origin <link do repositório> : Para apontar para um repositório online no GitHub

git clone <link do repositório> : Para enviar as mudanças locais para o repositório online

git push origin master: Para enviar as mudanças locais para o repositório online

git branch <nome da branch a ser criada>: Para criar uma nova branch

git checkout <nome da branch >: Para mudar de branch.

git merge <nome da branch a ser mergeada>: Para salvar alterações de uma branch separada na branch que estamos trabalhando

git diff:Para revisar modificações e comparar os aqruivos com o últimos commit feito. O git diff não poderá ser utilizado para arquivos novos, que ainda não estão sendo rastreados pelo Git (ou seja, que ainda não tiveram o primeiro git add executado). Para verificar um intervalo de commits utilizamos: git diff 222cccc..8877887 onde passamos os ids dos commits separados por dois pontos("..")

Utilizando o gitignore

Suponhamos que temos um arquivo que não queremos enviar ou salvar no nosso repositório.

Para isso utilizamos o gitignore, criamos um arquivo com o nome : ".gitignore", e dentro deste arquivo colocamos o nome e extensão do arquivo a ser ignorado, ex: "texto.txt", se precisamor ignorar todos os arquivos com a extensão txt, fazemos da seguinte forma: "*.txt", assim todos os arquivos com a extensão txt serão ignorados.

Há um projeto no GitHub com exemplos de arquivos .gitignore para diversas linguagens de programação e tecnologias: https://github. com/github/gitignore
